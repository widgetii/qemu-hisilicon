# QEMU HiSilicon CV300/EV300 Emulator

QEMU machine definitions for HiSilicon Hi3516CV300 and Hi3516EV300 IP camera SoCs,
targeting QEMU v10.2.0. Runs [OpenIPC](https://openipc.org/) firmware.

## Hardware Summary

| Feature | CV300 | EV300 |
|---------|-------|-------|
| CPU | ARM926EJ-S (ARMv5) | Cortex-A7 (ARMv7-A) |
| Interrupt ctrl | PL190 VIC @ 0x10040000 | GICv2: DIST 0x10301000, CPU 0x10302000 |
| RAM base | 0x80000000 | 0x40000000 |
| UART0 | 0x12100000, IRQ 5 | 0x12040000, SPI 7 |
| Timers | SP804 @ 0x12000000 | SP804 + ARM arch timer (50 MHz) |
| SysCtrl | 0x12020000 | 0x12020000 |
| CRG | 0x12010000 | 0x12010000 |
| GPIO | 9 × PL061 @ 0x12140000 | 10 × PL061 @ 0x120b0000 |

## Project Structure

```
qemu/
├── hw/arm/hisilicon.c           # Machine definitions (CV300 + EV300)
├── hw/misc/hisi-sysctl.c        # SysCtrl device (SoC ID, reboot)
├── hw/misc/hisi-crg.c           # CRG clock/reset stub
├── include/hw/arm/hisilicon.h   # Memory map constants
└── setup.sh                     # Clone QEMU v10.2.0, copy sources, build
qemu-boot/
├── run-cv300.sh                 # Run script for CV300
└── run-ev300.sh                 # Run script for EV300
qemu-src/                        # (generated by setup.sh, not committed)
```

## Quick Start

```bash
# Build
bash qemu/setup.sh

# Download OpenIPC firmware
cd qemu-boot
gh release download -R openipc/firmware -p "openipc.hi3516ev300-nor-lite.tgz"
tar xzf openipc.hi3516ev300-nor-lite.tgz

gh release download -R openipc/firmware -p "openipc.hi3516cv300-nor-lite.tgz"
tar xzf openipc.hi3516cv300-nor-lite.tgz

# Boot
bash qemu-boot/run-ev300.sh
bash qemu-boot/run-cv300.sh
```

Default login: `root` / `12345`

## Current Status

### EV300 — Fully Working

- Linux 4.9.37 boots to login prompt
- All core peripherals: GICv2, UARTs, timers, GPIOs, SPI, I2C, RTC
- OpenIPC userspace: syslogd, dropbear (SSH), crond, ntpd, majestic
- `ipctool` correctly identifies **HiSilicon 3516EV300**, reports RAM, kernel, firmware version
- Clean debug log (8 lines — PL022 extended regs and one sysctl offset)

### CV300 — Boots to Userspace

- Linux 3.18.20 boots, mounts rootfs, starts services
- `ipctool -c` correctly identifies **hi3516cv300**
- Full `ipctool` (with hardware probing) hangs due to slow ARM926 emulation (see below)
- Does not reach a login prompt with full init — hangs during vendor module loading (`mmz` allocator)

## Known Issues

### CV300: Slow MDIO/I2C Polling on ARM926

The `ipctool` binary probes Ethernet (MDIO) and sensor (I2C/SPI) hardware by polling
registers via `/dev/mem` with `usleep(1)` loops. On emulated ARM926EJ-S, each poll
iteration takes orders of magnitude longer than real hardware, causing the full
`ipctool` to hang for minutes.

**Workaround**: Use `initcall_blacklist=hisi_femac_driver_init,himci_init` in kernel
args (already set in `run-cv300.sh`) to disable the kernel-side Ethernet and MMC
drivers that cause similar polling timeouts during boot. The `ipctool -c` (chip
detection only) works instantly.

### CV300: No Login Prompt with Full Init

The OpenIPC init system hangs after loading vendor kernel modules (HiSilicon media
memory zone allocator). The `mmz` module likely needs hardware registers that aren't
emulated. With `init=/bin/sh` the system is fully usable.

### SysCtrl Stub Gaps

A few sysctl register offsets are accessed by the kernel but not implemented:
- CV300: offset 0x150 (write), 0x8c (read)
- EV300: offset 0x130 (read/write)

These are logged as guest errors but don't affect boot.

### CRG PLL Lock Bits

The CRG stub returns PLL lock (bit 28) for known PLL status register offsets
(0x04, 0x14, 0x24, 0x34). Other PLL registers may exist — add them if the kernel
hangs waiting for a PLL lock.

### No Flash Emulation

SPI NOR/NAND flash controller (FMC at 0x10000000) is not emulated. The kernel
probes it and fails gracefully. Rootfs must be loaded via `-initrd` as a ramdisk.
