# QEMU HiSilicon IP Camera SoC Emulator

QEMU machine definitions for HiSilicon IP camera SoCs (V1 through V4 generations),
targeting QEMU v10.2.0. Boots unmodified [OpenIPC](https://openipc.org/) firmware
to a full Linux userspace on all supported platforms.

## Supported Machines

| Machine | Generation | CPU | Kernel | Status |
|---------|-----------|-----|--------|--------|
| `hi3516cv100` | V1 | ARM926EJ-S | 3.0.8 | Full boot to shell |
| `hi3516cv200` | V2 | ARM926EJ-S | 4.9.37 | Full boot to shell |
| `hi3516cv300` | V3 | ARM926EJ-S | 3.18.20 | Full boot to shell |
| `hi3516ev300` | V4 | Cortex-A7 | 4.9.37 | Full boot to shell |
| `hi3516ev200` | V4 | Cortex-A7 | — | Config variant of EV300 |
| `hi3518ev300` | V4 | Cortex-A7 | — | Config variant (no on-chip PHY) |
| `hi3516dv200` | V4 | Cortex-A7 | — | Config variant (higher res) |
| `gk7205v200` | V4/Goke | Cortex-A7 | — | Rebrand of EV200 |
| `gk7205v300` | V4/Goke | Cortex-A7 | — | Rebrand of EV300 |
| `gk7202v300` | V4/Goke | Cortex-A7 | — | Rebrand of 3518EV300 |
| `gk7605v100` | V4/Goke | Cortex-A7 | — | Rebrand of DV200 |

```

├── hw/i2c/hisi-i2c.c            # HiBVT I2C controller
├── hw/i2c/hisi-imx335.c         # IMX335 image sensor I2C device

├── run-cv300.sh                 # Run script for CV300
└── run-ev300.sh                 # Run script for EV300
qemu-src/                        # (generated by setup.sh, not committed)
```

## Quick Start

```bash
# Build QEMU
bash qemu/setup.sh

# Download OpenIPC firmware (example: all 4 primary platforms)
cd qemu-boot
for soc in hi3516cv100 hi3516cv200 hi3516cv300 hi3516ev300; do
    curl -sL -O "https://github.com/OpenIPC/firmware/releases/download/latest/openipc.${soc}-nor-lite.tgz"
    tar xzf "openipc.${soc}-nor-lite.tgz"
done

# Download U-Boot binaries
for soc in hi3516cv100 hi3516cv200 hi3516cv300 hi3516ev300; do
    curl -sL -O "https://github.com/OpenIPC/firmware/releases/download/latest/u-boot-${soc}-universal.bin"
done
cd ..

# Boot any platform
bash qemu-boot/run-cv100.sh
bash qemu-boot/run-cv200.sh
bash qemu-boot/run-cv300.sh
bash qemu-boot/run-ev300.sh
```

Default login: `root` / `12345`

## Architecture

The emulator uses a table-driven design. Each SoC is a `HisiSoCConfig` struct
with addresses, IRQs, and peripheral counts. A single `hisilicon_common_init()`
function handles all platforms — adding a new SoC variant requires only a new
config struct and a one-line `DEFINE_HISI_MACHINE()` macro call.

### Three Memory Map Eras

| Era | SoCs | SysCtrl | UARTs | RAM | VIC/GIC |
|-----|------|---------|-------|-----|---------|
| Era 1 (V1/V2) | CV100, CV200 | 0x20050000 | 0x20080000 | 0x80000000 | VIC |
| Era 2 (V3) | CV300 | 0x12020000 | 0x12100000 | 0x80000000 | VIC |
| Era 3 (V4) | EV300, Goke | 0x12020000 | 0x12040000 | 0x40000000 | GICv2 |

### Appended DTB Auto-Patching

OpenIPC uImages for DT-based kernels (V2+) have an appended device tree with
zero padding and no `/chosen` node. The emulator automatically detects the
appended DTB, adds `chosen/stdout-path` for console output, and inserts padding
so the kernel's `atags_to_fdt()` can safely add initrd properties. This runs
transparently — unmodified firmware artifacts work as-is.

## SD Card Emulation

```bash
# Create a 64 MB image with FAT32
dd if=/dev/zero of=sdcard.img bs=1M count=64
echo -e "o\nn\np\n1\n\n\nt\nc\nw\n" | fdisk sdcard.img
LOOP=$(sudo losetup --find --show --partscan sdcard.img)
sudo mkfs.vfat -F 32 -n SDCARD "${LOOP}p1"
sudo losetup -d "$LOOP"

# Attach to any platform
bash qemu-boot/run-ev300.sh -drive file=sdcard.img,if=sd,format=raw
```

## Ethernet

The FEMAC device provides a single 8 KiB MMIO region with port, MDIO, and GLB
register blocks. An integrated PHY stub at MDIO address 1 reports 100 Mbps
full-duplex (PHY ID `0x00446161`). TX uses DMA read + `qemu_send_packet()`;
RX uses a 64-entry ring buffer with IRQ notification.

Run scripts pass `-nic user` for QEMU's SLIRP networking (NAT to host,
gateway `10.0.2.2`). CV100's hieth-sf driver gets DHCP on first boot;
CV200/300/EV300 FEMAC link-up takes a few seconds — DHCP may need a retry.
